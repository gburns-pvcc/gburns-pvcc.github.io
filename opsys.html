Operating Systems Terminology

TERM DEFINITION
operating system the software that controls the overall operation of a computer
job queue a storage organization in which jobs wait for execution
FIFO first-in, first-out: objects are removed from the queue in the order in which they arrived
interactive processing allows a program to carry on a dialogue with the user through remote terminals
time-sharing to provide service to multiple users at the same time
multiprogramming a technique in which time is divided into intervals and then the execution of each job is restricted to only one interval at a time
application software consists of the programs for performing tasks particular to the machine's utilization
system software provides the infrastructure that the application software requires
utility software consists of programs for performing activities that are fundamental to computer installations but not included in the operating system
graphical user interface (GUI) in which objects to be manipulated, such as files and programs, are represented pictorially on the display as icons
shell an interface that communicates with users through textual messages using a keyboard and monitor screen
kernel the internal part of an operating system containing software components that perform the very basic functions required by the computer installation
device driver the software units that communicate with the controllers (or at times, directly with peripheral devices) to carry out operations on the peripheral devices attached to the machine
file manager maintains records of all the files stored in mass storage, including where each file is located, which users are allowed to access the various files, and which portions of mass storage are available for new files or extensions to existing files
memory manager is charged with the task of coordinating the machine's use of main memory
virtual memory This large "fictional" memory space created by paging
boot strapping the procedure that transfers the operating system from mass storage (where it is permanently stored) into main memory (which is essentially empty when the machine is first turned on)
ROM read-only memory: its contents can be read but not altered
deadlock the condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another
spooling the technique of holding data for output at a later but more convenient time
super user (or administrator) a user that gains highly privileged access to the operating system by identifying him- or herself as the administrator (usually by name and password) during the login procedure
multi-factor authentication to require more than one piece of evidence to gain access
