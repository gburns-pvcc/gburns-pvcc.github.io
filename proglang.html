Programming Languages

TERM DEFINITION
machine language the collection of instructions encoded as bit patterns, along with the encoding system
debugging Locating bugs in the software and correcting them
assembler converts a mnemonic form program into machine language
assembly language a mnemonic system for representing programs is collectively
translator a program that translates programs expressed in high-level primitives into machine-language programs
compiler translators that compile several machine instructions into short sequences to simulate the activity requested by a single high-level primitive
interpreter a program that executes the instructions as they were translated instead of recording the translated version for future use
cross-platform software software that is independent of an operating system's design as well as the machine's hardware design and is therefore executable throughout a network
natural language evolved over time without formal grammatical analysis (English, German, and Latin)
formal language defined by grammars (programming languages)
machine independence Since the statements in a third-generation language did not refer to the attributes of any particular machine, they could be compiled as easily for one machine as for another. A program written in a third-generation language could theoretically be used on any machine simply by applying the appropriate compiler.
high-level programming language include many kinds of abstractions, ranging from the simple-constants, literals, and variables-to moderate complexity-statements, expressions, and control structures-to the esoteric-procedural units, modules, and libraries
object-oriented programming (OOP) a software system is viewed as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects
scripting language A subset of the imperative programming languages used to perform administrative tasks rather than to develop complex programs
data types:
  • integer - refers to numeric data consisting of whole numbers, probably stored using two's complement notation
  • float - refers to numeric data that might contain values other than whole numbers, probably stored in floating-point notation
  • string - strings of text using single quotation marks
primitive data type the data types that are included as primitives in a programming language, such as int for integer and char for character
data structure the conceptual shape or arrangement of data
Python collection data structures
  • list - an ordered, and changeable, collection
  • tuple - an ordered, and unchangeable, collection
  • set - an unordered, and unchangeable, collection
  • dictionary - an unordered, and changeable, collection
function a set of instructions for performing a task that can be used as an abstract tool by other program units
local variable a variable that can be referenced only within that function
global variable variables whose scopes are not restricted to a particular part of a program
source program the program in its original form
object program the translated version of the source program
